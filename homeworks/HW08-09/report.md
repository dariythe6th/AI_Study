# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

В работе исследуется влияние методов регуляризации и гиперпараметров оптимизации на обучение многослойной нейронной сети (MLP).

В **части A (регуляризация)** сравнивались:
- базовая модель
- модель с Dropout
- модель с Batch Normalization
- модель с Early Stopping

В **части B (оптимизация)** анализировалось влияние:
- слишком большого learning rate
- слишком маленького learning rate
- оптимизатора SGD с momentum и weight decay

**Цель** — определить конфигурацию, обеспечивающую наилучшую обобщающую способность модели.

## 4. Базовая модель и обучение

Использовалась многослойная нейронная сеть MLP.

- **Архитектура:**  
  `Input → Linear(256) → ReLU → Linear(128) → ReLU → Linear(10)`
- **Loss функция:** `CrossEntropyLoss`
- **Базовый оптимизатор:** Adam, `learning rate = 0.001`
- **Batch size:** 64
- **Максимальное число эпох:** 6 (для базовых экспериментов)
- **EarlyStopping:** `patience = 3`

---

## 5. Часть A (S08): регуляризация (E1–E4)

- **E1 (base)**  
  Базовая модель с двумя скрытыми слоями без регуляризации.
- **E2 (Dropout)**  
  К базовой модели добавлен Dropout(p=0.3) после скрытых слоёв.
- **E3 (BatchNorm)**  
  К базовой модели добавлены Batch Normalization слои.
- **E4 (EarlyStopping)**  
  Модель с Dropout обучалась с использованием EarlyStopping, который останавливает обучение при отсутствии улучшения validation loss.

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1–O3)

- **O1**  
  Adam с слишком большим learning rate (`lr = 0.1`).
- **O2**  
  Adam с слишком маленьким learning rate (`lr = 1e-5`).
- **O3**  
  SGD с momentum и weight decay:
    - optimizer: SGD
    - momentum = 0.9
    - weight_decay = 1e-4
    - lr = 0.1

- **Итоговая test accuracy:** ≈ 0.97–0.98

- **Что видно на O1 (слишком большой LR):**  
  Модель обучается нестабильно, loss снижается медленно и accuracy значительно ниже базовой модели.
- **Что видно на O2 (слишком маленький LR):**  
  Модель практически не обучается — loss почти не уменьшается, accuracy остаётся около случайного уровня.
- **Как повёл себя O3 (SGD + momentum + weight decay):**  
  SGD показал хорошую сходимость, но итоговая точность немного ниже, чем у Adam.

## 8. Анализ

По графикам обучения можно увидеть различия в динамике обучения моделей.

Базовая модель обучается стабильно, однако её точность ниже, чем у моделей с дополнительными методами регуляризации. Добавление **Dropout** немного снизило точность, что объясняется небольшим размером датасета: регуляризация уменьшает переобучение, но на малых данных может слегка ухудшать результат.

**Batch Normalization** показал наилучшие результаты. Это связано с тем, что нормализация активаций стабилизирует распределение входов для каждого слоя и ускоряет обучение. На графиках видно, что модель с BatchNorm быстрее снижает loss и достигает высокой accuracy.

**EarlyStopping** предотвращает переобучение, останавливая обучение при отсутствии улучшения validation loss. В данном случае обучение продолжалось до максимального числа эпох, так как метрика продолжала улучшаться.

Эксперименты с **learning rate** показали ожидаемые результаты. При слишком большом learning rate (O1) обновления параметров становятся слишком крупными, из-за чего оптимизация становится нестабильной. При слишком маленьком learning rate (O2) обновления параметров слишком малы, и модель обучается крайне медленно.

Использование **SGD с momentum** (O3) позволило добиться хорошей сходимости. Momentum помогает ускорять обучение в направлениях с устойчивым градиентом, а weight decay выполняет роль L2-регуляризации, уменьшая переобучение. Тем не менее Adam оказался более эффективным оптимизатором для данного датасета.

Выбранная лучшая конфигурация с BatchNorm является разумной для этого датасета, поскольку она обеспечивает быструю и стабильную сходимость при высокой точности классификации.
